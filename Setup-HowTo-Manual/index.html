CodeMAID User Manual

Terminal AI coding assistant · v3.0.0 · MIT License

CodeMAID is a privacy-first, local AI coding assistant that runs entirely on your machine. No telemetry, no cloud dependencies, no paywalls. It connects to local LLMs via Ollama or cloud APIs (OpenAI, Anthropic) and gives the AI access to your filesystem, shell, git, and the web — all sandboxed by the Litterbox security layer.

Installation

Prerequisites

  • Python 3.10 or newer
  • Ollama (for local models) — or an OpenAI / Anthropic API key for cloud
  • A model that supports tool calling, e.g. qwen3:14b, qwen2.5-coder:14b, llama3.1:8b

Install from the project directory

cd /home/cuckaoccurs/Projects/apps/CodeMAID
pip install -e . --break-system-packages

Verify

codemaid --help
# or the short alias:
codemaid --help
Tip: After any code change, re-run pip install -e . to pick up the update without reinstalling.

First Run

Start Ollama (if using local models)

ollama serve
# in another terminal:
ollama pull qwen3:14b

Launch CodeMAID

# Start in the current directory
codemaid .

# Start in a specific project
codemaid ~/Projects/myapp

# Non-interactive (pipe a prompt)
codemaid -p "what tests are failing?"

# Use a specific model
codemaid --model qwen2.5-coder:14b .

# Use a cloud provider
codemaid --provider openai --model gpt-4o .

First-run setup wizard

codemaid onboard

The onboarding wizard saves your preferred provider and model to ~/.config/codemaid/config.json.

Tool calling required: The AI uses tools to read/write files and run commands. Only models that support function/tool calling will work correctly. Older or smaller models may respond with text only and never use tools.

The TUI Interface

CodeMAID uses a natural-scroll terminal UI. New output appears at the bottom and scrolls the terminal upward — no special terminal capabilities needed.

Layout

  [conversation history — banner at top, newest messages at bottom]
  ──────────────────────────────────────────────────────────
  ❯ your input here                                          ← input line
  ──────────────────────────────────────────────────────────
  🐱 /path/to/dir   │  litterbox: on (denylist)   │  model   ← status bar

Chatting with the AI

Just type and press Enter. The AI has access to all configured tools and will use them automatically as needed.

KeyAction
EnterSubmit message
BackspaceDelete last character
Ctrl+CExit CodeMAID
Ctrl+C mid-responseInterrupt the AI (then type to continue)

File Tools

All file tools are restricted to the working directory by path confinement. Absolute paths and ../ traversals outside the working directory are blocked.

ToolWhat it does
read_fileRead a single file (truncated at 20 KB)
write_fileCreate a new file or overwrite an existing one
edit_fileSEARCH/REPLACE block edit — precise in-place change; falls back to fuzzy match
read_multipleRead multiple files in one call
list_dirList files and directories with sizes
diff_previewShow unified diff of last edit to a file
undo_editRestore file from automatic pre-edit backup
edit_planPropose a multi-file edit plan before executing (used in plan mode)
Automatic backups: Every file edit creates a timestamped backup in ~/.config/codemaid/backups/. Use undo_edit to revert or diff_preview to inspect changes. Backups rotate automatically after 500 files.

Litterbox Security Sandbox

Every command passed to run_command is validated before execution through three layers:

  1. Denylist — blocks known-dangerous patterns
  2. Path confinement — file tools cannot access paths outside the working directory
  3. Firejail — optional container sandbox (used automatically if firejail is installed)

Blocked by default

  • Filesystem destruction: rm -rf /, dd if=/dev/zero, mkfs
  • Pipe-to-shell: curl ... | sh, wget ... | bash
  • Credential theft: cat ~/.ssh/id_rsa, cat /etc/shadow
  • Shell spawning: sh, bash, zsh as commands (not as file extensions)
  • Code injection: eval, base64 -d, python -c
  • Reverse shells, SSRF, cloud metadata endpoints

Modes

ModeCommandBehaviour
Denylist (default)/litterbox + /allowlistBlock known-dangerous patterns, allow everything else
Allowlist/allowlistOnly permit explicitly safe commands (ls, cat, git, pytest, etc.)
Off/litterboxNo validation — use only in a fully trusted environment
Warning: Turning the litterbox off (/litterbox) removes all command safety checks. Only do this if you fully trust the model and understand the risks.

Path Confinement

All file tools resolve paths to their real location (following symlinks) and verify the result is inside the working directory before proceeding. This prevents:

  • Path traversal attacks (../../etc/passwd)
  • Symlink escape attacks
  • Suffix-matching bypasses (/project-evil/ when workdir is /project/)

The audit log is written to ~/.config/codemaid/audit.log — every file read, write, edit, and command execution is recorded as a JSON line.

Troubleshooting

SymptomCauseFix
Streaming error: 400 Bad Request Model doesn't support tool calling, or Ollama is outdated Use a tool-capable model: /model qwen3:14b or qwen2.5-coder:14b. Update Ollama.
Ollama not reachable Ollama server isn't running Run ollama serve in another terminal
AI never uses tools Model doesn't support function calling Switch to a model that supports tools (see Providers table above)
🛡️ LITTERBOX BLOCKED Command matched a dangerous pattern Review the command. If legitimate, use /litterbox to disable temporarily.
Access denied: outside workdir Path confinement blocked the path Start CodeMAID from a higher directory, or use codemaid /the/target/dir
Context window exceeded warning Conversation history too large Run /compress or /subcompress to trim history
codemaid command not found Renamed to codemaid in v3 Use codemaid or codemaid
Audit log: All tool actions are logged to ~/.config/codemaid/audit.log as JSON lines. Tail it with !tail -f ~/.config/codemaid/audit.log from inside CodeMAID to watch live activity.
